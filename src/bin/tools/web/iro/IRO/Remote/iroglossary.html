IRO Observers Guide

IRO Schedule: Glossary of Terms

Observer's code

The observer's code is a three letter string assigned by the IRO administrator. Observers' codes can be requested by submitting a request for an observing code. Astronomy students at the University of Iowa are assigned codes by the lab instructor.

Project Title

The project title is short phrase describing the project. The title is written as a COMMENT to the FITS header.

Observer Name(s)

Enter your name(s). This is written as a COMMENT to the FITS header.

  • Do not put quotes or apostrophes in the title or observer names!

Filter and Duration

Identify the desired filter and exposure duration, in seconds, in the two fields provided.

Multiple values may be entered by separating each entry with a comma. Filter/durations entries are paired; if one field has more entries than the other the right-most value of the deficient field is reused as often as necessary.

For example, to take two images of a source, one through the V filter for a 30 second exposure and one through the R filter for 60 seconds, enter the fields as: 

	V,R
	30,60
The telescope has a motorized filter wheel. Nine filters are currently supported for remote observing:
  • Clear filter (code C): Recommended for general purpose deep sky images.
  • Johnson photometric filters (codes B, V, R, and I). These are 100 nm bandpass filters centered at 450,550,650, and 850 nm.
The exposure time of an image depends on several factors, including filter selection, apparent magnitude of the object, and sky conditions. For clear skies and low zenith angle (high elevation angle), the following table can be used to estimate the maximum exposure time to avoid saturation.

App. Mag. C B V R I
2 - 0.1 0.05 0.05 0.1
4 0.05 0.5 0.25 0.25 0.5
6 0.5 3.5 1.5 1.5 3.5
8 2 21 10 10 22
10+ 12 60 60 60 60

Object Name

This field allows you to name the observation.

If the position of the object is given using the Coordinates method then the name may be anything whatsoever. If the position is being derived from a Catalog the name must be exactly as it appears in the catalog (but case and whitespace are ignored). Examples are given in each catalog selection. As a special case, asteroids may be entered by number only, e.g. 4 instead of Vesta 4.

The Object Name is used to set the OBJECT field of the FITS header.

  • The orbital elements of many solar system objects are already known, so the program will calculate the celestial coordinates at the time of observation. Only uncataloged objects need celestial coordinates entered by the observer.

Celestial Coordinates

The celestial coordinates describe the position of the source in the sky. The format is hh:mm:ss for the right ascension and dd:mm:ss for the declination, with a preceding sign when appropriate. The epoch of the coordinates is in years.

LST or HA Start Time

Observations are normally scheduled automatically as close to the meridian as possible given all the other observing requests being processed in a given night. However, if it is important to observe your object at a specific local Sideral Time or Hour Angle then specify the time in the given field and set the radio button to indicate which time is being utilized. The format of the time field is hh:mm:ss. Leave the field entirely blank if you do not wish to specifiy a start time.

If the scheduling program cannot schedule the observation within 30 minutes of the requested time, it is not scheduled at all.

Compression Scale Factor

This requests compression of the image using the H Compression algorithm. The Space Telescope Science Institute offers the H compression algorithm for UNIX and VMS. The University of Iowa's ATFtools includes the H compression algorithm for Windows 95.

Lossless compression (scale 1) typically results in a factor of 2-3 in image size reduction, while a factor of 100 results in a factor of 6-8. Compression using more than a factor of 100 is not recommended, especially for photomteric analysis. A value of 0 requests no compression.

Compressed files will be named *.fth; uncompressed files will be named *.fts.

Repeat Parameters

An observation (including one which involves multiple filters) may be observed at regular intervals throughout a night. This option is useful for generating a sequence of images for light curves of asteroids, variable stars, etc.

Set the first field to the number of total observations. Set the second field to the elapsed time desired between observations in the format H:M:S. Due to a variety of factors, the actual elapsed time between the resulting observations may vary from the value requested.

Comment Line

You may enter a text string (no quotes or apostrophes) in this box. This will be written to the FITS header as a COMMENT keyword.
